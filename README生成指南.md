# Prompt 文档索引生成指南

## 概述

本文档描述了如何从一个包含多个 prompt 文件的目录，生成一个结构清晰、便于查阅的索引文档（README.md）。

## 生成流程

### 第一步：扫描源目录

识别源目录中的所有文档文件，获取文件列表。

### 第二步：分析文档内容

对每个文档进行内容分析，提取以下关键信息：

| 提取项 | 说明 |
|--------|------|
| 功能描述 | 该 prompt 的核心用途 |
| 适用场景 | 什么情况下应该使用此 prompt |
| 核心内容 | 该 prompt 涵盖的主要内容点 |

### 第三步：设计索引结构

采用统一的模板组织信息：

```markdown
### 序号. 文档名称

**功能**: 简要描述

**适用场景**: 
- 场景一
- 场景二

**核心内容**:
- 内容点一
- 内容点二

**使用方式**: 阅读原文档了解详情
```

### 第四步：生成索引文档

按以下结构组织最终文档：

1. **项目标题与简介**：说明文档集合的用途
2. **文档列表**：每个文档的摘要信息
3. **快速选择指南**：帮助用户快速找到所需文档

### 第五步：添加导航辅助

在文末添加决策表格，帮助用户根据需求快速定位：

| 需求 | 推荐文档 |
|------|----------|
| 具体需求描述 | 对应文档名称 |

## 设计原则

### 1. 信息层次清晰

- 使用标题层级区分结构
- 使用加粗、列表等格式突出重点
- 保持各文档描述格式一致

### 2. 内容适度抽象

- 不直接复制原文内容
- 提炼核心要点
- 保留原文链接供深入阅读

### 3. 便于快速检索

- 提供功能定位
- 明确适用场景
- 添加决策辅助表格

### 4. 保持可维护性

- 结构化模板便于更新
- 新增文档时按模板添加即可
- 格式统一减少维护成本

## 模板参考

```markdown
# [项目名称] 文档索引

本项目包含多个文档，用于[项目用途说明]。

## 文档列表

### 1. [文档名称]

**功能**: [功能描述]

**适用场景**: 
- [场景一]
- [场景二]

**核心内容**:
- [内容点一]
- [内容点二]

**使用方式**: 阅读 [原文档](链接) 了解详情

---

## 如何选择

| 需求 | 使用文档 |
|------|----------|
| [需求描述] | [文档名称] |
```

## 扩展建议

1. **版本管理**：可在文档列表中添加版本号和更新日期
2. **标签系统**：为文档添加标签便于分类检索
3. **依赖关系**：说明文档之间的关联或前置依赖
4. **示例链接**：如有示例代码，可添加链接
